# 상단 네비게이션 갭 문제 수정

## 개요
각 페이지 상단에 불필요한 빈 공간이 있어 네비게이션이 뷰포트 최상단에 붙지 않는 문제 해결. 원인은 `padding-top: var(--header-height)`가 적용되어 있으나 실제 글로벌 Navbar 컴포넌트가 없었기 때문.

## 주요 변경사항
- 수정한 것: 14개 페이지의 CSS에서 불필요한 `padding-top: var(--header-height)` 제거
- 추가 수정: `pageNav` 컴포넌트의 `top` 값을 `0`으로 변경

## 수정된 파일 목록

### Ranking 페이지
- `src/app/ranking/total/page.module.css`
- `src/app/ranking/vip/page.module.css`
- `src/app/ranking/vip/[userId]/page.module.css`

### Info 페이지
- `src/app/info/org/page.module.css`
- `src/app/info/live/page.module.css`
- `src/app/info/sig/page.module.css`
- `src/app/info/timeline/page.module.css`

### Notice 페이지
- `src/app/notice/page.module.css`
- `src/app/notice/[id]/page.module.css`

### Schedule 페이지
- `src/app/schedule/page.module.css`

### Community 페이지
- `src/app/community/free/page.module.css`
- `src/app/community/[category]/[id]/page.module.css`

### Components
- `src/components/ranking/RankingList.module.css` (CSS 파싱 에러 수정: `var(--space-3.5)` → `var(--space-3)`)

## 핵심 코드
```css
/* Before */
.main {
  padding-top: var(--header-height);
}
.pageNav {
  top: var(--header-height);
}

/* After */
.main {
  /* padding-top 제거 */
}
.pageNav {
  top: 0;
}
```

## 결과
- ✅ 모든 페이지에서 네비게이션이 뷰포트 최상단에 붙음
- ✅ 메인 페이지 글로벌 Navbar 정상 작동
- ✅ 개별 페이지 자체 pageNav 정상 작동
- ✅ CSS 파싱 에러 해결
